# Cloud Concepts answer key

1. **A — IaaS.** It preserves control of the guest OS and supports lift-and-shift scenarios.
2. **B — PaaS.** The provider manages the OS and platform while developers focus on the application.
3. **C — SaaS.** The requirement is a complete, ready-to-use application.
4. **C — Scale up.** The size of one existing resource increases.
5. **A — Scale out.** The number of instances increases.
6. **C — Hybrid.** The solution combines connected on-premises and public-cloud resources.
7. **B — The customer.** The customer remains responsible for identities, access decisions, and organizational data.
8. **A — Within the same region.** Zones provide physical separation and independent infrastructure inside a region.
9. **C — No.** Region pairing does not automatically replicate every resource.
10. **C — Management group.** It provides a scope above subscriptions for governance at scale.
11. **B — No.** A resource belongs to one resource group at a time.
12. **A.** The tenant is the identity directory; the subscription is a management, billing, quota, and resource boundary.
13. **B — Elasticity.** Capacity follows temporary demand, often automatically.
14. **C.** Deleting the group deletes its resources. The resources can use different regions, and groups cannot be nested.
15. **A — Azure Functions/serverless.** Event-driven execution without server administration is the key clue.

## Score guide

- **13–15:** Strong. Explain every wrong option before moving on.
- **10–12:** Review the comparisons behind the missed questions.
- **0–9:** Revisit the concept pages, then retry without memorizing answer positions.

A score from a repeated quiz can measure memory rather than understanding. Use different practice sets and explain the distractors aloud.
